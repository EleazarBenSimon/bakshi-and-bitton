# Security Policy

## Reporting a vulnerability or sensitive data issue

If you discover a security vulnerability, a data-integrity bug that could be exploited, a data-subject privacy concern, or a factual correction request, please contact the maintainer privately:

**Email**: `eleazarbensimon@gmail.com`

Public Issues are currently disabled on this repository to keep moderation manageable. All correction and privacy requests should be sent via email; we aim to respond within 7 business days.

Please include in your report:

1. A clear description of the issue
2. Steps to reproduce, if applicable
3. Any sensitive context that should not appear in the public record

The maintainer will acknowledge receipt within 7 days and provide an estimated timeline for response.

## Scope

This policy covers:

- Security issues in the website or any tooling under `scripts/`
- Errors in dataset entries that could expose individuals to demonstrable harm
- Data-subject access, correction, or deletion requests under applicable Israeli or international privacy law (see [About](site/about.html))

## What this project does NOT consider security issues

- Disagreement with the methodology — please open a public issue per [CONTRIBUTING.md](CONTRIBUTING.md)
- Political disagreement with a ruling's outcome — out of scope; the dataset records facts, not opinions
- Requests to remove factual entries that link to public-record court rulings — not vulnerabilities; the rulings are public

## Public-record posture

This project publishes structured metadata about publicly-published Israeli Supreme Court rulings, with every claim linked back to the official ruling text on supreme.court.gov.il. Inclusion in the dataset is itself not a privacy harm under Israeli or international law for public-figure justices and for petitioners whose petitions are matters of public record.

If you believe a specific entry contains personal data outside the public-record exception, please use the email above. Such requests will be evaluated on the merits.
